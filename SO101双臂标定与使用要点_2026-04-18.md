## 最重要的规则

- `port` 是当前串口，只表示设备接在哪个 `/dev/ttyACM*`
- `id` 决定读取哪个标定文件
- LeRobot 读取标定文件看的是 `id`，不是 `port`

结论：

- `/dev/ttyACM0` 不会天然等于 `0.json`
- `/dev/ttyACM1` 不会天然等于 `1.json`
- 必须在代码里显式把端口映射到正确的 `id`

## 如果端口顺序变了

Linux 重新插拔设备后，`ttyACM0` 和 `ttyACM1` 可能互换。

如果发现：

- `/dev/ttyACM0` 实际是右臂
- `/dev/ttyACM1` 实际是左臂

就把这 7 个脚本里的映射改成：

```python
PORT_TO_ROBOT_ID = {
    "/dev/ttyACM0": "bimanual_right",
    "/dev/ttyACM1": "bimanual_left",
}
```

否则会出现：

- 左臂加载右臂标定
- 右臂加载左臂标定

## 串口权限

当前设备节点权限例如：

```bash
ls -l /dev/ttyACM*
```

如需把当前用户加入串口组，执行：

```bash
sudo usermod -a -G dialout $USER
newgrp dialout
```

如果执行后仍有权限问题，最稳妥的方法是：

- 退出当前终端重新登录一次

## 标定命令写法

多行命令时，每一行结尾都要带 `\`。

正确示例：

```bash
lerobot-calibrate \
  --robot.type=bi_so_follower \
  --robot.id=bimanual \
  --robot.left_arm_config.port=/dev/ttyACM0 \
  --robot.right_arm_config.port=/dev/ttyACM1
```

## 关于 wrist_roll

- `wrist_roll` 不出现在 `MIN / POS / MAX` 表里是正常的
- 它按整圈关节处理
- 它不参与手动录制最小值和最大值
- 它使用固定范围 `0 ~ 4095`
- 但它仍然参与第一步“摆到中位”的偏置标定

## 给后来使用者的最短结论

- 先确认左臂和右臂当前分别对应哪个 `ttyACM`
- 再确认这 7 个示例脚本里的端口映射是否和实际一致
- 当前默认规则就是：
  - `ACM0 -> bimanual_left.json`
  - `ACM1 -> bimanual_right.json`
