# 1️⃣ XLeRobot ACT 左右臂标定

> 默认只手动标定 **左臂 6 电机 + 右臂 6 电机**。
> 第一次接入硬件、动过零位、换电机后必做。

---

## 🎯 标定的目的

让每个电机知道：
- **零位**（home pose）：所有关节角 = 0 时是什么物理姿态
- **行程**（range）：每个关节能走的最小/最大角度

不标定 → 主从方向反、行程对不上、归一化失效，必出问题。

---

## 📁 标定文件位置

ACT 标定脚本会把左右臂标定写到两轮版机器人标定目录：
```
~/.cache/huggingface/lerobot/calibration/robots/xlerobot_2wheels/<follower_id>.json
```

`<follower_id>` 默认值是 `xlerobot_main`（在 [act/config.py](../../act/config.py) 里设置）。

---

## 🛠️ 二、运行标定

```bash
python act/calibrate_xlerobot.py
```

如确实要走官方整机标定，再显式加：
```bash
python act/calibrate_xlerobot.py --full-body
```

### 流程（终端会顺序提示）

当前 ACT 默认流程不会要求你手动标头部和底盘；头部/底盘字段会从已有标定或电机当前参数补齐。

**bus1（左臂 6 个电机）**
1. 把**左臂**摆到中位姿态 → 回车
2. 依次推动**左臂**非连续关节覆盖任务范围 → 回车

**bus2（右臂 6 个电机；两轮底盘自动补齐）**
3. 把**右臂**所有关节摆到行程中段 → 回车
4. 把**右臂**所有关节挨个推到极限位 → 回车

> ✅ **头部 2 电机和两轮底盘 2 电机不需要你手动标定**。
> ⚠️ 标定时电机扭矩会被自动关掉，所以推动很轻松；推完后系统会自动把扭矩拉回来。

## ♻️ 四、什么时候要重标

| 场景 | 是否重标 |
|---|---|
| 第一次接入 | ✅ |
| 换电机 / 拆装关节 | ✅ |
| 改了 `--robot.id` / `FOLLOWER_ID` | ✅（新名字对应新文件） |
| 长时间没用、怀疑零位漂 | 建议 |
| 只是换 USB 端口 | ❌ |

---

## ⚠️ 五、常见标定坑

| 现象 | 原因 | 解决 |
|---|---|---|
| 关节卡住推不动 | 扭矩没松 / 物理障碍 | 断电再上电；移开障碍 |
| 标完跟随方向反 | 极限位推反了 | 重标，注意先推到正方向再回零 |
| 标完行程缩了一半 | 没推到真极限 | 重标，听到电机微响再停 |
| 启动时被问 ENTER 还是 c | XLeRobot 询问是否复用旧标定 | 直接回车 = 用旧标定；输入 `c` 回车 = 重标 |
